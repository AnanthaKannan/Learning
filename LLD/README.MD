## What is the use of static methods in a class?
Static methods belong to the class itself rather than to any specific instance of the class. They can be called without creating an instance of the class.
## What is the use of abstract class?
* In TypeScript, an abstract class is a special kind of class that cannot be instantiated directly ‚Äî you can‚Äôt create an object from it.
* Define a common structure that multiple subclasses can share.
* Enforce a contract ‚Äî i.e., ensure all subclasses implement certain methods.
```ts
abstract class Animal {
  // Abstract method (must be implemented by subclasses)
  abstract makeSound(): void;

  // Concrete method (shared logic)
  move(): void {
    console.log("Moving...");
  }
}

class Dog extends Animal {
  makeSound(): void {
    console.log("Woof! üê∂");
  }
}

class Cat extends Animal {
  makeSound(): void {
    console.log("Meow! üê±");
  }
}

const dog = new Dog();
dog.makeSound(); // Output: Woof! üê∂
dog.move();      // Output: Moving...

// const animal = new Animal(); ‚ùå Error: Cannot create an instance of an abstract class
```
## What is Polymorphism?
Polymorphism is a core concept in object-oriented programming that allows objects of different classes to be treated as objects of a common superclass. It enables a single interface to represent different underlying forms (data types). The most common use of polymorphism is when a parent class reference is used to refer to a child class object.
```ts
class Animal {
  makeSound(): void {
    console.log("Some generic animal sound");
  }
}
class Dog extends Animal {
  makeSound(): void {
    console.log("Woof! üê∂");
  }
}
class Cat extends Animal {
  makeSound(): void {
    console.log("Meow! üê±");
  }
}

function playSound(animal: Animal): void {
  animal.makeSound();
}

const myDog = new Dog();
const myCat = new Cat();
playSound(myDog); // Output: Woof! üê∂
playSound(myCat); // Output: Meow! üê±
```
